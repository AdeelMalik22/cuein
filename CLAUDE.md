# Working Rules

- Build this project one small, working step at a time.
- Before changing code, explain the immediate goal in plain language and wait for the user's direction when the next step changes scope.
- Prefer simple, explicit code over abstractions that hide how multi-tenancy works.
- Keep tenant ownership visible in models, queries, permissions, and tests as the app grows.
- Make and verify one focused change at a time.
- Never create a document about the implementation. Keep implementation guidance in the code, tests, and this conversation unless the user explicitly changes this rule.
- Do not create files beyond those the user has requested.
